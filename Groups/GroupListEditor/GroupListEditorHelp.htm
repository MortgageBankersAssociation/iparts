Group List Editor

  • Name - Enter a name for the content item. This text appears during design mode (configuration).
  • Title - Enter the text to display above this iPart on the rendered page. If blank, no title displays.
  • Do not render in design mode - Select this option to prevent content record data from being loaded in the content editor. This improves performance during design time (configuration).
  • Display a border around this content - Select this option to display a border around this iPart on the rendered page.
  • Display for authorized users only - (enabled by default) Hides the iPart from public users when accessing data for another person, which prevents spoofing (changing the ID in the URL to access another's data). Disable this option if you need to make this iPart visible to public users, such as for a public profile.
  • Display for authorized users only - (enabled by default) Hides the iPart from public users when accessing data for another person, which prevents spoofing (changing the ID in the URL to access another's data). Disable this option if you need to make this iPart visible to public users, such as for a public profile. Note: This iPart may behave differently if you have edited the permissions defined in your ClaimsMap.xml and ClaimTemplates.xml files. By default, only System Administrators and members of the Staff Group have the claims to access data for any user in the system.
  • Display profile image - Select this option to display profile images as thumbnails in the group list.
  • Show inactive memberships - Select this option to display both active and inactive members within a group list.
  • Allow editing - Select this option to allow users to add, edit, or delete members within any group listed. Some restrictions apply: 1) Only System Administrators and Staff (Full and Casual users) can add members to a group. 2) Only group members, System Administrators, and Staff can delete members from a group and edit membership information if available.
  • Button text for adding a contact - Enter the text to display on the button a user clicks to add group members.
  • Mode - Select a mode to specify how you want the iPart to function. The options available below the Mode section change depending on which mode you select. In each mode, the iPart uses the value of a URL parameter to generate the group list displayed to the user. By default, the iPart looks for a UniformKey or iUniformKey parameter value. If neither is available, the iPart uses the selected or logged in user as the parent ID. You can override the default behavior by specifying an alternate URL parameter that the iPart should use to generate the group list. 
    • Display a group's members and any sub-groups - Select this mode to list a group's members and any sub-groups, if they exist. For example, enable this mode to display all members of a chapter and any committee groups within that chapter.
      • URL parameter for group ID - Leave this field blank, unless you want to change which URL parameter the iPart uses for a group ID value.
      • Choose the group types to include - The selected group types determine which sub-groups are displayed in the initial list and in expanded contact nodes. For example, if a chapter has both committee and club sub-groups, you can select one or both to be displayed within the iPart.
    • Display groups by parent ID - Select this mode to list groups owned by a particular parent ID. If multiple groups are owned by the parent ID, initially only the groups are listed, but each group can be expanded to display members. If only a single group is owned by the parent ID, initially all of the group members are listed. For example, if you add the Group List Editor to an organization contact's account page, it will automatically display an organization roster (a group member list). For example, if you add the Group List Editor to a Chapter's page, it will automatically list all of the groups owned by the Chapter, such as committees and clubs. But if the Chapter only has members and owns no other groups, then initially all of the Chapter members are displayed.
      • URL parameter for parent ID - Leave this field blank, unless you want to change which URL parameter the iPart uses for a parent ID value.
      • Choose the group types to include - The selected group types determine which groups are displayed in the initial list and in expanded contact nodes.
    • Display a single group type (will auto-create a group if one does not exist) - Select this mode to only display groups of a single type. If a group of that type does not already exist, users can automatically create a new group by adding a member. For example, you can choose to only display a chapter's committees. If no committees exist yet, no groups are listed, but a user can click add to automatically create a committee and add members. When this mode is selected, the configuration option Allow editing is enabled, so that groups can be automatically created. Select this mode if you want to display an organization roster, so that the organization group can be auto-created when you add the first member.
      • URL parameter for parent ID - Leave this field blank, unless you want to change which URL parameter the iPart uses for a parent ID value.
      • Parent type - Depending on how your system is set up, the available parent types can include broad categories such as Events, Contacts, Groups, and Public.
      • Selected group type - The selected group types determine which groups are displayed in the initial list and in expanded contact nodes.
    • Display a contact's memberships - Displays a list of memberships for individual and organization contacts.
      • URL parameter for contact - Leave this field blank, unless you want to change which URL parameter the iPart uses for a contact value. 
      • Choose the group types to include - Only memberships of the selected group type are displayed.

Notes

  • A good use for this iPart is to add it to a page that already displays information about a contact, event, or some other type of group page. In most cases, the Group List Editor will detect the group, parent, or contact ID in the URL and display a group list in the mode you select. If you add the iPart to a group page, and a group list does not display as you would expect, check the URL of the web page to determine if it uses a URL parameter other than UniformKey or iUniformKey.
  • You can create, edit, and delete group types, roles, and statuses from Tools > Setup > Groups.
  • You can hard code URL parameter values using a navigation item. From Site designer > Manage sitemaps, create or edit a navigation item and in the URL Parameters field, enter a parameter name and value. For example, ID=123.

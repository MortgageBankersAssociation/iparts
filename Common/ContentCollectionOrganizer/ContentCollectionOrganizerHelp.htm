Content Collection Organizer

  • Name - Enter a name for the content item. This text appears during design mode (configuration).
  • Title - Enter the text to display above this iPart on the rendered page. If blank, no title displays.
  • Do not render in design mode - Select this option to prevent content record data from being loaded in the content editor. This improves performance during design time (configuration).
  • Display a border around this content - Select this option to display a border around this iPart on the rendered page.
  • Function as wizard - Select this option to display the iPart as a workflow (wizard). If not selected, the iPart displays tabs for content items. If this option is selected, the following item displays.
  • Steps must be sequential (available if Function as wizard option is selected) - Select this option if your workflow has fixed sequential steps.
  • Display style - Select one of the following to specify the style of display:
    • Horizontal Top - Displays content defined for each tab/step in horizontal tabs across page.
    • Numbered Steps - (available if Function as wizard option is selected) Displays each step with Step Name and Step x of y at the top of the content record defined for each step.
    • Unnumbered Steps - (available if Function as wizard option is selected) Displays each step with only the Step Name at the top of the content record defined for each step.
  • Click + to add a tab/step.
  • Enter the Tab/Step Name to display.
  • Browse for Content to display. When selecting a content record to display, keep in mind that access to content records can be limited by security. To prevent errors, make sure that the content records you select can be accessed by the intended users. Access Settings are defined when editing a content record from Content designer > Manage content.
  • Click Preview to check the data to be displayed.
  • Next step on success - (available if Function as wizard option is selected and Steps must be sequential option is not selected) Select the step to display if the current step is successful.
  • Next step on failure - (available if Function as wizard option is selected and Steps must be sequential option is not selected) Select the step to display if the current step fails. Default = remain on the current step.
  • Hide this step - (available if Function as wizard option is selected) Select this option to hide the step to the user (does not display). This allows you to attach any action iParts without user intervention. For example, you might want to send an email message when a certain step is complete.
  • Move left - Click to move the selected tab/step to the left (changes display/process order).
  • Move right - Click to move the selected tab/step to the right (changes display/process order).

Notes

  • When displaying multiple Content Collection Organizer (CCO) iParts on a single web page, you must configure each CCO iPart to have a unique Name.
  • You can create links to specific tabs of your CCO iPart using URL parameters. The parameter is either passed through from the webpage's navigation item or from a direct link to the page, and it can be built using either the tab name or tab index. The available URL parameters are [iPart Name]=[Tab name] and [iPart Name]=[tab index]. For example, if you want to create a link directly to the Giving History tab of your Account Pages CCO iPart, set the URL to http://mywebsite.com/Account_Page.aspx?Account_Pages=Giving History. Or, if you want to attach that same parameter to the navigation item instead, enter Account_Pages=Giving History into the navigation item's URL Parameters field. You also have the option to link by the tab index. For example, if the Giving History tab is the second tab displayed, use Account_Pages=2.

    When creating a link to a tab, keep the following guidelines in mind:

    • Use the iPart Name and Tab name defined within the configuration page of your CCO iPart.
    • Replace any spaces in the iPart Name with underscores.
    • Enter the exact Tab name, including spaces.
    • Direct tab linking does not work when the option Function as wizard is enabled.
    • If you change the iPart Name, Tab name, or tab order, you may need to update any navigation items or links that point to that tab.

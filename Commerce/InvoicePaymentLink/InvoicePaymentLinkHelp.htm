Invoice Payment Link

  • Name - Enter a name for the content item. This text only appears during design mode (configuration).
  • Title - Enter the text to display above this iPart on the rendered page. If blank, no title displays.
  • Do not render in design mode - Enable this option to prevent content record data from being loaded in the content editor. This improves performance during design time (configuration).
  • Display a border around this content - Select this option to display a border around this iPart on the rendered page.
  • Button text - Enter the text to display on the button users click to add invoices to their cart.
  • Message - Enter text to be displayed above the button. For example, you could enter a message that explains to users what to expect when they click the button.
  • Choose the types of invoices to include
    • Standard - Standard invoices include items such as open invoices for customer purchases.
    • Renewal - Renewal invoices include unpaid membership fees and subscriptions.
  • Display options when no invoices are due - Select the option for how the iPart is displayed when no open invoices are due.
    • Hide button and Message - When no invoice is due, the button and Message are hidden.
    • Show disabled button and No open invoice message - When no invoice is due, a disabled button and the No open invoice message are displayed.
  • No open invoice message - Enter a message to be displayed when no invoice is due.

Notes

  • The iPart displays a conditional button to pay an outstanding renewal invoice. When the user clicks the button, the renewal item is placed into the cart. Users do not have the option to selectively pay for individual line items within the invoice.
  • The button displays only if the signed in user has an unpaid renewal invoice that is not already in the cart for items such as membership fees or subscriptions.
  • Note for organizations that process renewal billings and use accrual-based accounting: Currently, this iPart can only be used to pay for the most recently billed renewal invoice. Unpaid invoices for previously billed renewal billings cannot be paid with this iPart. This may be an issue if your organization has closely-recurring billing cycles, or if some of your users renew multiple memberships through your organization and those memberships have different billing cycles.
  • The button displays only if there is an outstanding invoice that is not already in the cart for one of the following contacts:
    • The signed in user
    • The contact currently selected as On Behalf Of
    • The contact whose record is being viewed (the button is disabled)
    • The button is enabled only if the contact with the outstanding invoice is signed in directly or On Behalf Of. Clicking the button adds the renewal to the cart of that contact. If a page containing a contact key in the URL displays a contact that is neither signed in nor selected for On Behalf Of, then the button is disabled. For example, if Smith signs in and views the account for Jones (who has an outstanding renewal), then the button is disabled until Smith acts on behalf of Jones.
  • If users have more than one unpaid invoice, all invoices are added to the cart. Users can select which invoices they want to pay from the cart.
